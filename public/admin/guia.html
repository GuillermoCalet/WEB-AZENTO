Editar la web con tranquilidad

El gestor permite cambiar contenido sin tocar el diseño. En producción, ninguna modificación de una clienta aparece en la web hasta que un administrador la revisa y aprueba.

Fíjate en el indicador inferior del panel. En modo local, guardar escribe en los archivos de este ordenador. En producción, los cambios siguen el flujo de borrador y revisión.
1

Edita

Abre una sección, cambia solo los campos que necesites y revisa los textos alternativos de las imágenes.

2

Guarda el borrador

Puedes volver a abrirlo y corregirlo tantas veces como necesites antes de enviarlo.

3

Envía a revisión

El gestor crea una propuesta separada. La web publicada todavía no cambia.

4

Espera la aprobación

El administrador revisa la vista previa, pide correcciones o aprueba la publicación.

Tareas habituales

  1. Cambiar un texto: entra en «Editar la web», abre la página correspondiente, modifica el campo y guarda.
  2. Sustituir una imagen: pulsa «Elegir imagen», selecciona o sube una nueva y completa su texto alternativo.
  3. Añadir un proyecto: abre «Divisiones, proyectos y servicios», elige la división y añádelo en la lista de proyectos.
  4. Reordenar una galería: arrastra los elementos de la lista. El orden del panel es el orden de la web.
  5. Ocultar contenido: desactiva «Visible» o «Publicado». Es preferible ocultar antes que borrar.

Imágenes

  • Formatos admitidos: JPG, PNG y WebP.
  • Tamaño máximo: 3 MB por archivo.
  • Dimensiones: máximo 8000 píxeles por lado y, para fotografías de contenido, al menos 300 píxeles por lado.
  • El texto alternativo debe describir lo que se ve; no uses nombres como «foto1».
  • No subas HEIC/HEIF cambiando solamente su extensión.

Qué no debes modificar

No cambies rutas, identificadores internos, enlaces técnicos o datos legales sin hablar antes con el administrador. El CSS, los tamaños, las posiciones y la estructura no aparecen en el CMS precisamente para proteger el diseño.

Pruebas locales

  1. Ejecuta npm run dev.
  2. En otra terminal, ejecuta npm run cms:dev.
  3. Abre http://localhost:4321/admin/index.html.
  4. Haz una copia o un commit antes de probar si quieres descartar fácilmente los cambios.
  5. Para recuperar archivos, pide al administrador que los restaure con Git; no borres carpetas manualmente.
